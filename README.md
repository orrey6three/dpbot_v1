# 🤖 DPS45 Telegram Bot

Этот бот слушает сообщения в группе и автоматически создает метки на карте [DPS45](https://ваша-ссылка.vercel.app).

## 🚀 Деплой (Deployment)

Поскольку бот должен работать постоянно, его нельзя разместить на Vercel. Мы рекомендуем использовать **Railway**, **Render** или любой **VPS**.

### 1. Подготовка
Убедитесь, что у вас есть:
- `TG_TOKEN` (токен от @BotFather)
- `OPENROUTER_API_KEY` (ключ для ИИ)
- `YANDEX_MAPS_API_KEY` (ключ для геокодинга)
- `API_URL` (URL вашего бэкенда на Vercel, обычно `https://ваше-приложение.vercel.app/api`)

### 2. Запуск на сервере
Склонируйте репозиторий, перейдите в папку бота и запустите его:

```bash
cd bot_v1
npm install
node bot.js
```

### 3. Переменные окружения (.env)
Создайте файл `.env` (или добавьте в настройки хостинга) следующие параметры:

| Переменная | Описание |
|------------|----------|
| `TG_TOKEN` | Токен бота от Telegram |
| `BOT_TOKEN` | Общий секретный ключ (должен совпадать с бэкендом) |
| `API_URL` | Адрес вашего API (например, `https://dps45.vercel.app/api`) |
| `CHAT_ID` | ID чата или @username группы (где бот будет слушать) |
| `OPENROUTER_API_KEY` | Ключ для OpenRouter (Gemini) |
| `YANDEX_MAPS_API_KEY`| Ключ Яндекс.Карт для поиска координат |

## 🛠 Как связать с бэкендом?
Бот отправляет данные на ваш Vercel по адресу `${API_URL}/patrol`. 
Убедитесь, что в настройках Vercel (в Dashboard) и в настройках бота указан **одинаковый** `BOT_TOKEN`.

## 🧠 Особенности
- **AI-парсинг:** Использует Gemini 2.0 для понимания сленга (типа "хпп", "монетка", "виадук").
- **Цепочки ответов:** Бот помнит контекст. Если ответить "стоят" на сообщение с адресом, он поймет, где именно стоят.
- **Авторство:** Все метки на карте будут подписаны вашим ником из Telegram.
